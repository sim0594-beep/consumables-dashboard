# 소모품 운영현황 대시보드 - SaaS 통합형 v2

풀무원아카데미 소모품 운영현황용 Next.js 대시보드입니다.

## v2 주요 변경

- 기본 접속 시 바로 `소모품 운영현황` 화면 표시
- 사이드바 메뉴 단순화
  - `소모품 운영현황`
  - `관리자 접속`
- 사이드바 상단 CI만 표시, 하단 텍스트 제거, 심볼 제거
- 관리자 접속은 비밀번호 입력 후 진입
- 관리자 화면 내 `현황 조회 / 업로드·수정 관리` 전환 바 삭제
- 관리자 모드 선택 3개만 유지
  - `종합정보관리`: 미화/의약품 + 푸드 업로드 모두 표시
  - `미화의약품관리`: 미화/의약품 업로드만 표시
  - `푸드관리`: 푸드 업로드만 표시
- 작동하지 않는 사이드바 버튼 제거

## 로컬 실행

```bash
npm install
npm run dev
```

브라우저에서 접속:

```text
http://localhost:3000
```

## 관리자 비밀번호 테스트

프로젝트 루트에 `.env.local` 파일을 만들고 아래처럼 입력합니다.

```env
ADMIN_PASSWORD=1234
```

`.env.local`이 없으면 테스트 기본값으로 `1234`를 사용합니다.

환경변수를 변경한 뒤에는 서버를 다시 실행하세요.

```bash
Ctrl + C
npm run dev
```

## Supabase 연결

Supabase를 연결하지 않으면 브라우저 저장모드(localStorage)로 테스트됩니다.
연결 시 `.env.local`에 아래 값도 추가합니다.

```env
NEXT_PUBLIC_SUPABASE_URL=본인_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_ANON_KEY=본인_SUPABASE_ANON_KEY
```

`supabase/migrations/001_consumables.sql`을 Supabase SQL Editor에서 실행하고, Storage bucket `consumable-files`를 생성하세요.


## v3 수정 사항

- `현황 한눈에 보기`의 `사용 품목 수`를 TOP 10 개수가 아니라 선택 월의 실제 사용 품목 수로 계산하도록 수정했습니다.
  - 기준: 당월 사용수량이 0보다 크거나 사용금액이 0보다 큰 품목
- `재고 보유 품목`을 TOP 10 개수가 아니라 선택 월의 실제 재고 보유 품목 수로 계산하도록 수정했습니다.
  - 기준: 현재 재고수량이 0보다 크거나 재고금액이 0보다 큰 품목
- 사용금액 TOP 10, 재고금액 TOP 10도 금액/수량이 0인 품목은 제외하도록 정리했습니다.


## v4 보완 사항

- 사이드바 버튼 색상을 v2처럼 식별 가능한 네이비/블루 톤으로 보정했습니다.
- 비활성 버튼도 흰색 배경으로 뜨지 않도록 명시적으로 배경/글자색을 지정했습니다.
- `6. 세부현황` 표에 `장소` 컬럼을 추가했습니다.
- 회사 CI 적용은 이미지 파일(PNG/SVG 권장)을 전달받은 뒤 사이드바 브랜드 영역에 삽입할 수 있습니다.


## v5 수정 사항
- 사이드바 상단 `PA CENTER` 텍스트를 업로드된 풀무원아카데미 CI 이미지로 교체했습니다.
- CI 하단의 `소모품 운영 시스템` 문구는 삭제했습니다.
- 사이드바 버튼 색상은 v2 스타일처럼 비활성/활성 상태가 명확히 구분되도록 되돌렸습니다.
- 세부현황 표의 `장소` 컬럼은 유지됩니다.


## v6 추가 반영
- CI 하단 `소모품 운영 시스템` 텍스트 삭제


## v7 반영사항

- 좌측 사이드바 제거, 상단 고정 헤더 구조로 변경
- 본문 최대 폭을 조정해 중앙 정렬 보고서형 레이아웃 적용
- KPI 아이콘 크기 20% 확대
- 월별 사용금액 추이를 가로형 메인 그래프로 확대 배치
- 분류별 사용금액 비중/주요 사용품목 영역 삭제
- 월간 수불 현황, 월별 사용금액 비교의 비고란 제거 및 표 폭 최적화
- 사용금액/재고금액 상위 10개 품목을 상하 배치하고 각 섹션 우측에 도넛 그래프 추가
- 현황 요약을 품목별 세부 현황 상단으로 이동
- 품목별 세부 현황에서 분류, 규격 컬럼 삭제 및 장소 컬럼 유지
- 데이터 관리 로그인 박스 중앙 정렬
- 최근 업로드 이력과 검증 리포트를 상하 배치
- 그래프 최초 진입 및 새로고침 시 애니메이션 적용
- 막대그래프 색상 기준으로 미화·의약품/푸드 캡슐 색상 계열 통일

## v8 반영사항

- 관리자 접속 화면 상단의 중복 제목 영역을 제거하고, 비밀번호 입력 카드를 상단 중앙에 배치했습니다.
- 운영현황/데이터 관리 페이지의 카드 간격을 14px 기준으로 정돈했습니다.
- `미화·의약품만 보기`, `푸드만 보기`에서는 월별 사용금액 추이 그래프의 `합계액` 막대와 범례를 제거했습니다.
- 그래프 하단 월별 합계액 별도 표기는 추가하지 않고 기존처럼 유지했습니다.
- 사용금액 상위 10개 품목과 재고금액 상위 10개 품목을 좌우 2열 배치로 되돌렸습니다.
- TOP 10 영역의 막대그래프와 도넛그래프를 제거하고, 품목명·캡슐·금액 중심의 리스트로 정리했습니다.
- TOP 10 품목명은 한 줄 유지 및 말줄임표 처리로 전체보기에서 줄바꿈이 생기지 않도록 조정했습니다.
- 품목별 세부 현황 표에서 수량 컬럼은 가운데 정렬, 금액 컬럼은 우측 정렬로 변경했습니다.
- 관리 화면의 업로드 카드 간격을 정리하고 데이터 검증 요약은 업로드 카드 하단에 배치했습니다.

## v9 refinement
